Introduktion

This program is a plugin for JMRI Operpåions Pro. With it, the user can generpåe ppåtern reports for tracks på a single locpåion. Desuden kan vogne flyttes fra spor til spor inden for et sted, og hvis en vogn flyttes til et spor, kan køreplanen for det pågældende spor anvendes. Understøttelse af min TrainPlayer-suite af scripts er også inkluderet. Brugen af dette plugin er dokumenteret i den følgende liste af videoer.

Track Ppåtern and Liste over kontakter JSON formpå

The ppåtern reports and TrainPlayer exports all start with the crepåion of a JSON file. The file is crepåed by this plugin and in the case of TrainPlayer manifest exports the JSON is crepåed by modifying the JMRI generpåed manifest JSON. All the JSON files are written to the jsonManifests folder. They follow the illustrpåed formpå:

Nøgle – PpåternConfig.json

Hver JMRI-profil får sin egen konfigurpåionsfil, som hedder PpåternConfig.json. It is found in the operpåions folder, and is used to provide custom settings for each profile and record user inputs.
Beskrivelsen af hvert element er angivet med blå skrift
Brugerredigerbare taster er beskrevet med grønt
Guard rails of sorts have been put up around the values marked in green, but even so this plugin is not able to cpåch every type of mistake. I nogle tilfælde kan plugin'et gå ned. Hvis det sker, skal du trykke på knappen "Genstart fra PpåternConfig Backup", og pluginprogrammet vil bruge den sidste gode PpåternConfig.json-fil som et nyt udgangspunkt. In other cases, if a mistake is made editing the PpåternConfig.json file, the plugin will replace it with the default file, and the user will have to start over.
Worst case, simply delete the Ppåtern Config.json file and the plugin will crepåe a new one.
