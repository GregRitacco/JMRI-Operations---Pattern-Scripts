Plugin Scripts de motifs

Introduction

Le plugin Pattern Scripts peut être considéré comme un cadre pour héberger des sous-programmes pour JMRI Operations Pro qui nécessitent une interface utilisateur. Ce plugin est livré avec quatre sous-programmes : jPlus, Patterns, o2o et Throwback. Un modèle est également inclus pour servir d'échafaudage pour quelqu'un qui souhaite ajouter son propre sous-programme. Bien que les sous-programmes soient quelque peu interdépendants, ils peuvent être activés ou désactivés individuellement selon les besoins.

configFile.json

Chaque profil JMRI dispose de son propre fichier de configuration, appelé configFile.json, qui se trouve dans le dossier des opérations et qui est utilisé pour fournir des paramètres personnalisés pour chaque profil et enregistrer les entrées de l'utilisateur.

La description de chaque élément est en rouge foncé.

Les clés modifiables par l'utilisateur sont décrites en vert.

Sélectionnez Tools/Edit Config File. Si une erreur est commise lors de la modification du fichier configFile.json, le plugin supprimera le fichier et en fournira un nouveau pour recommencer. Dans le pire des cas, le plugin peut se planter. Si cela se produit, sélectionnez Tools/Restart From Default.

Sous-programme jPlus

Description

Le sous-programme jPlus ajoute des détails ferroviaires améliorés aux rapports générés par JMRI et ce plugin.

Les données de cette sous-routine sont saisies dans les zones de texte ou, si la sous-routine o2o est activée, les détails supplémentaires du chemin de fer sont importés à partir du modèle TrainPlayer.

Tous les champs ne doivent pas être saisis.

Les champs améliorés sont les suivants :

Nom du chemin de fer en exploitation

Il s'agit du chemin de fer qui possède ou exploite les voies modélisées. N'importe quel nom peut être saisi, tel que "PRR" ou "BN UPRR Joint" ou "G&D".

Territoire opérationnel

Il s'agit du nom du chemin de fer pour la partie à modéliser, comme "Long Beach Intermodal Terminal" ou "Salem Glass Works Branch".

Localisation

L'endroit où se trouve le chemin de fer, par exemple "The Great Divide" ou "The Bronx, New York".

Année de fabrication

L'année sur laquelle le modèle ferroviaire est basé, c'est un miroir de 'JMRI/OperationsPro/Settings/Year Modeled'.

o2o Sous-routine

Description

La sous-routine ops to ops (o2o) est un groupe d'utilitaires conçus pour être la colle qui permet à JMRI Operations Pro d'être le moteur d'opérations pour TrainPlayer©. L'idée est d'étendre les capacités de la fonction Advanced Ops de TrainPlayer© pour inclure la flexibilité d'Operations Pro, tout en éliminant l'ennui de maintenir deux bases de données synchronisées.

Sous-routine Patterns

Description

La sous-routine Patterns permet de générer des rapports de modèle pour les voies d'un même emplacement. En outre, les wagons peuvent être déplacés d'une voie à l'autre au sein d'un emplacement. En option, si un wagon est déplacé sur un embranchement, le programme de cet embranchement peut être appliqué.
