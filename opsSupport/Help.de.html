Pattern Scripts plugin

Introduction

The Pattern Scripts plugin can be thought of as a framework for hosting subroutines for JMRI Operations Pro that require a user interface. This plugin comes with four subroutines; jPlus, Patterns, o2o and Throwback. A template is also included to serve as scaffolding for someone wishing to add their own subroutine.

Although the subroutines are somewhat interdependent, they may be added or removed individually as needed.

Additionally, each subroutine can be activated or deactivated as needed, fron the Tools menu pulldown.

Each of the subroutines' features are only available when the subroutine is active and the Pattern Scripts window is open. For example, extended railroad details will only appear in reports when the Pattern Scripts window is open and the jPlus subroutine is displayed. This is an easy way to turn features on or off.

jPlus Subroutine

Description

The jPlus subroutine adds enhanced railroad detail to reports generated by both JMRI and this plugin. This detail is displayed as the header for manifests and switch lists. The header for the JMRI server is not changed.

How To Use

Data for this subroutine is entered into the text boxes or if the o2o subroutine is activated, the additional railroad detail is imported from the TrainPlayer layout.

To use this subroutine. enter the appropriate data into the text box and click update. Not all fields need to be entered. The enhanced fields are as follows:

Operating Railroad Name

This is railroad that owns or operates the tracks being modeled. Any name can be entered such as 'PRR' or 'BN UPRR Joint' or 'G&D'.

Operational Territory

This is the railroad's name for the part being modeled, such as 'Long Beach Intermodal Terminal' or 'Salem Glass Works Branch'.

Location

Where the the railroad is located, such as 'The Great Divide' or 'The Bronx, New York'.

Year Modeled

Which year the model railroad is based on, this is a mirror of 'JMRI/OperationsPro/Settings/Year Modeled'. Either updates the other.

o2o Subroutine

Description

The ops to ops (o2o) subroutine is a group of utilities that are designed to be the glue that allows JMRI Operations Pro to be the operations engine for TrainPlayerÂ©. The idea is to expand the capability of TrainPlayerÂ©'s Advanced Ops feature to include the flexibility of Operations Pro, while also eleminating the tedium of keeping two databases synchronised.

o2o will automatically export a built train or switchlist to TrainPlayer when the Pattern scripts window is open and the o2o subroutine is displayed. To disable o2o export, disable o2o from the Tools pulldown or close the Pattern Scripts window.

What the buttons do

Before any of these buttons are pressed, export your layout from TrainPlayer. The QuickKeys for TrainPlayer subroutines have to be added to TrainPlayer before your TrainPlayer layout can be exported. Refer to the Quick Keys documentation.

The Quick Keys page on GitHub

New JMRI Railroad - Pressing this button erases any existing data and builds a new JMRI railroad from the Advanced Ops window via a TrainPlayer export. The Routes and Trains XML files are backed up, the others are discarded.

Update Locations - Pressing this button updates your JMRI railroad to the changes made to the TrainPlayer Advanced Ops Locations and Industries tabs, along with any changes made to your TrainPlayer rolling stock.

Update Industries - Pressing this button only applies changes made to the TrainPlayer Advanced Ops Industries tab. Nothing else is updated. This effectively updates only JMRI schedule data.

Update Rolling Stock - Pressing this button applies changes made to your TrainPlayer rolling stock to your JMRI rolling stock. If a change is made to existing cars, the final destination is not overwritten.

Patterns Subroutine

Description

The Patterns subroutine can generate pattern reports for tracks at a single location. Additionally, cars can be moved from track to track within a location. Optionally, if a car is moved onto a spur, the schedule for that spur can be applied.

What the buttons do

If you are using divisions, the Division drop down will display a list of divisions. Selecting a division will load a list of locations for that division into the Location drop dow.

The Location drop down will display a list of locations for your layout, or just the locations assigned to the selected division.

With a location selected, a list of check boxes will display, one for each track at the selected location.

The list of tracks can be filtered by clicking the 'Yard Tracks Only' check box on or off.

Individual tracks can be selected by clicking the check box next to the track name.

With tracks selected, pressing the Pattern Report button will display inventory information about the selected tracks. This is printable.

With tracks selected, pressing the Set Rolling Stock to Track button will pop open one worklist for each track selected.

For each worklist window, a row of buttons, one for each track of the selected location, will display. Pressing one of the track buttons will load the track name into memory.

For any of the rolling stock rows in the Cars At... panel, clicking inside of one of the text boxes will paste the selected track name into the box.

Pressing the Switch List button will generate a switch list of work for that track. This is printable. If o2o is being used, a TrainPlayer work list will also be generated.

Pressing the Set Rolling Stock to Track button will open a pop up.

The pop up adds the option to use a spurs schedule and overide track capacity.

Pressing the Set Cars button will cause the rolling stock in the worklist to be moved to the selected track, subject to type restrictions.

Throwback plugin

Description

The Throwback Subroutine is modeled on the idea of version control software. Commits are named and saved. In so doing snapshots are taken of the railroads data. The commit then serves as a setpoint that allows any of the JMRI data sets to be returned to.

What the buttons do

The name of each commit is entered into the text box of the Total Commits frame. Enter the name BEFORE pressing the Add New Commit button.

The Previous and Next buttons facilitate scrolling through your collection of commits if more than one commit has been created. The list is ordered by creation date/time.

Pressing the Reset All Commits button erases all saved commits, so be careful.

In the Actions panel, select any combination of data sets to be thrown back by clicking its check box.

Pressing the Throwback button sets the selected data set to the setpoint displayed in the Total Commits text box.

This subroutine is called Throwback, but any of your data sets can be thrown to any commit, back or foreward.

This subroutine does not interact with JMRIs Auto Save or Auto Backup feature, they run as configured.
